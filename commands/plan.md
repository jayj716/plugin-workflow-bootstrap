# /plan — 아이디어 기획 파이프라인

막연한 제품 아이디어를 대화형으로 구체화해 **PRD → 기능명세서 → 유저플로우 → 와이어프레임**을 순서대로,
서로 정합성을 유지하며 만든다. (manyfast "Manny" 흐름을 Claude Code로)

## Dispatch

1. 같은 플러그인의 번들 스킬 지침을 그대로 읽고 따른다: `skills/idea-to-plan/SKILL.md`
   (및 그 `references/` 전부)
2. 사용자가 `/plan` 뒤에 아이디어를 적었으면 그것을 0단계 입력으로 사용한다.
   비어 있으면 먼저 한 문장 아이디어를 물어본 뒤 인터뷰(0단계)부터 시작한다.
3. 단계마다 멈춰 사용자 확인을 받는다. "한 번에 다 해줘"라고 하면 연속 진행한다.
4. 기획이 끝나고 구현으로 넘어갈 때는 `idea-to-plan`의 5단계(OpenSpec 핸드오프)를 따르거나,
   `agentic-coding-workflow` 스킬로 이어간다.
